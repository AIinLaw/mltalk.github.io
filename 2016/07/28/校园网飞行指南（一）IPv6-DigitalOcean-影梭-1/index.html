CUPL网费贵么？大概不贵，一个月10元/20G流量还算便宜。网络带宽也蛮良心，千兆网口也能让你的下载速度飞起来。如果你是一个“正常”的网络用户，大概不需要下面的折腾。如果现状足够满足你的需求，请不要在这里浪费时间；如果你不嫌麻烦且希望有些突破，那么就听我絮叨絮叨。

声明： 教程会尽可能详细，但挂一漏万在所难免，有任何问题请不要忘记先Google一下。同是互联网小白，作者不负有给任何人答疑解惑的义务，分享仅源于互联网之精神。

目标：利用DigitalOcean提供的VPS搭建影梭（shadowsocks），实现利用IPv6访问国际互联网，顺带实现墙裂，自由访问需要的网站。

工具：Xshell、 DigitalOcean账号、notepad++等

创建VPS服务


点击Creat Droplet，然后对下面选项依次设置：

1.Choose an image
大致意思是选择一个操作系统，其实差别不太大，本教程及选择的CentOS系统。即：
Distributions —— CentOS 7.2 X64

2.Choose a size
选择第一个最便宜的5$/月即可，据说价钱越贵速度越快，流量是1T但是后台并没有流量统计工具，所以用超了也好像没啥事儿，但滥用会遭到封号，不要太过即可。

3.Choose a datacenter region
选择服务器所在的位置,推荐旧金山（San Francisco）节点。延迟小、状态比较稳定

4.Select additional options
其他选项：勾选IPv6选项



其他选项默认即可，然后在页面底部点击Create，便自动创建一个VPS。

安装shadowsocks程序


VPS创建好后，你的邮箱会收到一封邮件，里面有你的VPS的相关信息。类似下图:




打开XShell软件，在中间黑框内输入：
ssh root@IP Address

即邮件中的IP Address

然后弹出提示，密码填邮件中的password。这是第一次登录VPS，会要求你改密码，按操作进行即可：输入一遍当前的密码，回车；输入你的新密码，回车。


注意：linux下输密码时屏幕不带回显，连星号也不显示，小心输入后大胆回车即可


然后依次输入下面两行指令，输完一条按一下回车，中间遇到提示直接按y通过。
yum install python-setuptools && easy_install pip
pip install shadowsocks

此时shadowsocks已经安装在了你的VPS上。

配置shadowsocks


然后依次执行以下三条指令：
cd /etc
touch shadowsocks.json
vi shadowsocks.json

此时按 i 键进入编辑模式，XShell黑框的左下角会出现 ­­–INSERT–­­ 字样，然后一次性复制下面的内容到notepad++之类的文本编辑器里，修改password，这个密码就是你以后shadowsocks登录的密码），然后右键粘贴到软件里面。
{
    “server”:”::”,
    “server_port”:443,
    “local_address”: “127.0.0.1”,
    “local_port”:1080,
    “password”:”mypassword”,
    “timeout”:300,
    “method”:”aes-256-cfb”,
    “fast_open”: false
}


复制完成后，按 Esc 键退出编辑模式，此时XShell黑框左下角的 ­­–INSERT–­ 字样消失，按下 : 键，输入 wq 后回车，此时文件保存完毕并退出了vi编辑器。

提醒：这里千万不要将复制的代码粘贴到windows的记事本中修改后粘贴过来，建议使用notepad++之类的文本编辑器，不然会出现编码错误。


然后在xshell输入：
cd

回车即回到服务器根目录。

配置IPv6

登录digital ocean网站

双击你创建好的droplet，然后选择左侧边栏中的 Networking

你将在页面最底端看到如下ipv6信息：


Alt text
这时，在Xshell中依次输入下面两条命令，并回车：

ip -6 addr add public_ipv6_address/64 dev eth0
ip -6 route add default via public_ipv6_gateway dev eth0
图中的**public_ipv6_address**和**public_ipv6_gateway**就是图片中对应打马的具体内容 注意：PUBLIC IPV6 ADDRESS那一栏的ipv6地址，digital ocean有时会在尾部加上了/64. 复制时不要把/64复制进去，即`/64`只出现一次。 到这里就把ipv6配置完了。

启动shadowsocks服务

用下面的指令可分别在后台启动或者停止shadowsocks服务： 
ssserver -c /etc/shadowsocks.json -d start
ssserver -c /etc/shadowsocks.json -d stop
至此，基本的服务端配置就结束了，可以正常使用了。 注意：如果在启动shadowsocks服务时报错，显示“Expecting，Delimiter...”之类的报错语句，是因为我前面所说的你用的windows记事本或其他类似的软件修改配置参数造成字符格式不对。可以换其他编辑器试试。

shadowsocks客户端设置

**与上面的配置一一对应：** 
- 服务器IP即 上面的ipv6地址或者ipv4地址


- 服务器端口(server_port)：默认443


- 密码是上面设置中的 mypassword


- 加密方式（method）: aes-256-cfb


- 代理端口（local_port）：1080
